公司介绍

Shopee是东南亚和台湾地区领先的电子商务平台，隶属于Sea集团，总部位于新加坡。作为一个电商平台，Shopee为用户提供了一个便捷的购物环境，用户可以通过移动端应用和网页版浏览和购买各类商品。

自2015年首次亮相以来，Shopee已经在东南亚和台湾地区迅速扩张，并已在新加坡、马来西亚、泰国、台湾、印度尼西亚、越南和菲律宾开展业务。2018年，Shopee还进入了巴西市场，2021年进入了印度、波兰、西班牙、法国等市场，标志着其全球扩张战略的继续。

项目展示

配置页面优化（第一期：4.23）
Agency Type Priority 页面
• 实现 Agency 优先级配置功能，支持可视化设置。字段：Agency 类型、优先级权重、生效时间。
BAU Driver Blacklist 页面
• 配置仅服务于 Campaign 的司机黑名单。功能：支持批量导入/导出、模糊搜索、状态筛选。
Driver-Ops Configuration 页面
• 记录 Ops 与司机的负责关系，支持按区域分配任务。交互：下拉框动态加载可用司机，已分配的司机灰显。
Non-BAU Management 页面
• 配置非日常揽收日期（如节假日、Campaign 日）。联动业务规则，禁止在配置日期生成揽收任务。
Vehicle Configuration 页面
• 配置车辆类型、容量、通行限制（如大型车辆）。支持按 Cluster 维度批量操作（SEA 无此功能）。

调度管理 & 动态 CT 组（第二期：5.10）
FM Cluster Polygon 配置
• 基于地图的 Cluster 区域划分，支持多边形绘制与编辑。 校验规则：Cluster 间不可重叠，需符合车辆通行限制。
Scheduler 页面
• 创建/更新 Pickup Task，支持动态路由参数配置。 展示任务状态（待计算、已分配、已完成）。
Dynamic CT Group 功能
• CT Group 列表页：展示计算任务组，支持按状态、Cluster 筛选。
• CT 详情页：复用 SEA 结构，新增 BR 字段（如 Toll Fee、Potential Missed PUP）。
• 地图操作页： 拖拽调整路线节点，校验规则（如车辆数超限时禁止拖拽）。 展示 BR 特有数据：距离（保留 1 位小数）、费用（保留 2 位小数）。
• 新增功能：Required Vehicle Num 输入、异常订单标记（Check Issue）。

Fixed CT & Backup CT（第三期：5.23）
Fixed CT 页面（新增）
功能描述
• 创建 Fixed CT：支持手动创建固定计算任务（Fixed CT），指定 Cluster、时间窗、车辆类型等参数。
• Split/Merge CT Group：
• • Split：将一个 CT Group 按规则（如区域、订单量）拆分为多个子任务。
• • Merge：合并多个 CT Group，优化路径规划与司机负载均衡。
• Route ID 继承：
• • 为首次生成 Trip 时创建 CIOT 通行证，确保后续 Backup 或 Periodic 任务继承同一 Route ID，避免重复计费。
交互设计
• 地图操作页：
• • 拖拽调整路线节点，校验规则：
• • • 若 vehicle_num > 1，禁止手动修改路线。
• • • 跨 Cluster 拖拽时，弹出 Toast 提示：“当前操作可能导致路径超限，请确认！”
• • 展示字段：
• • • 距离（保留 1 位小数）、费用（保留 2 位小数）、预计耗时。
• • 新增功能：
• • • Required Vehicle Num：输入指定车型数量，系统自动校验资源可用性。
• • • Check Issue：标记异常订单（如地址错误、包裹超限），高亮显示并同步至 Ops 处理。
 Backup CT 页面
功能描述
• 手动插入 Trip：支持 Ops 在突发情况（如车辆故障、订单激增）时手动插入备用揽收任务。
• 同步至 FM Schedule：
• • 接口字段对齐：确保新增 Trip 的 Route ID、时间窗、费用明细完整传递至下游系统。
• 费用分摊规则展示：
• • 按 Trip 维度展示成本分摊比例（如司机工资、高速费、CIOT 通行证费用）。
• • 数据来源：对接 Finance 系统 API，实时更新费用明细。
交互设计
• 列表页：
• • 支持按状态筛选（待分配、进行中、已完成）、按 Cluster 分组。
• • 批量操作：一键分配司机、导出任务明细。
• 详情页：
• • • 复用 SEA 结构，新增 BR 字段：
• • • • Toll Fee：展示路段高速费，联动地图路径实时计算。
• • • • Potential Missed PUP：标记可能遗漏的揽收点，提供“强制分配”选项。
